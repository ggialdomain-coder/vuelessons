Step 1: Understanding Dynamic Components

Goal: Create a tab system where clicking tabs switches between different components (HomeTab, ProductsTab, ContactTab).

What are Dynamic Components?

  • ‚úÖ Allow you to switch between different components dynamically
  • ‚úÖ Use <component :is="componentName"> syntax
  • ‚úÖ Perfect for tabs, modals, multi-step forms
  • ‚úÖ Component name comes from data property

How it works:

  • You have multiple components (HomeTab, ProductsTab, ContactTab)
  • You track which one to show in data: activeTab: 'HomeTab'
  • You use <component :is="activeTab"> to render it
  • When you change activeTab, Vue automatically switches components!
üí° Key Concept: The :is attribute tells Vue which component to render. When it changes, Vue unmounts the old component and mounts the new one.

Step 5: Set Up Data Properties

What we're doing: Create data to track tabs and active tab.

Key concept: We need an array of tab component names and a property to track which one is active.

data() { return { tabs: ['HomeTab', 'ProductsTab', 'ContactTab'], activeTab: 'HomeTab' } }

Explanation:

  • tabs - Array of component names (used for buttons with v-for)
  • Component names must match exactly: 'HomeTab', 'ProductsTab', 'ContactTab'
  • activeTab - Current active tab component name
  • Default is 'HomeTab' - shows HomeTab first when page loads
  • When user clicks a tab button, we'll change activeTab to that tab's name

Step 6: Register Components

What we're doing: Register all tab components so Vue knows about them.

Key concept: Components must be registered before they can be used with :is.

components: { HomeTab, ProductsTab, ContactTab }

Explanation:

  • Register all three tab components in the components option
  • Component names in components must match the names in tabs array
  • Vue will use these registered components when :is references them
  • If component name doesn't match, Vue won't be able to find it!

Step 7: Create Tab Buttons with v-for

What we're doing: Create buttons for each tab using v-for to loop through the tabs array.

Key concept: Loop through tabs array and create a button for each tab, with active highlighting.

<div class="tabs"> <button v-for="tab in tabs" :key="tab" :class="{ active: activeTab === tab }" @click="activeTab = tab" class="tab-button"> {{ tab }} </button> </div>

Explanation:

  • v-for="tab in tabs" - Loops through tabs array, creates a button for each tab
  • :key="tab" - Required for v-for (unique identifier for each button)
  • :class="{ active: activeTab === tab }" - Adds 'active' class when this tab is the active one
  • When activeTab === tab is true, the button gets the 'active' class (highlighted)
  • @click="activeTab = tab" - When button clicked, sets activeTab to that tab's name
  • {{ tab }} - Displays tab name on button (HomeTab, ProductsTab, ContactTab)

Step 8: Add Dynamic Component

What we're doing: Use <component :is> to render the active tab component.

Key concept: The component changes automatically when activeTab changes.

<div class="tab-content"> <component :is="activeTab"></component> </div>

Explanation:

  • <component :is="activeTab"> - Renders component based on activeTab value
  • When activeTab = 'HomeTab' ‚Üí renders HomeTab component (shows home info)
  • When activeTab = 'ProductsTab' ‚Üí renders ProductsTab component (shows products list)
  • When activeTab = 'ContactTab' ‚Üí renders ContactTab component (shows contact info)
  • Vue automatically switches components when activeTab changes!
  • No manual DOM manipulation needed - Vue handles everything!
üí° Magic: When you click a tab button, activeTab changes, and Vue automatically unmounts the old component and mounts the new one. The content switches instantly!
