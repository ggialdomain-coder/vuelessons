Step 1: Understanding What We're Building

Goal: Create a Card component with named slots (header, body, footer) that can accept different content.

What we need:

  • Card component with default slot
  • Named slots: header, body, and footer
  • Styled slots (header with green background, footer with light background)
  • Default content for each slot

What are Slots?

  • âœ… Slots allow you to pass HTML content to components

Step 2: Set Up the Card Component Structure

What we're doing: Create the basic Card component structure.

Key concept: Start with the outer card div and prepare for slots.

app.component('Card', { template: ` <div class="card"> <!-- Slots will go here --> </div> ` });

Explanation:

  • Create the Card component with app.component()
  • Template has a <div class="card"> wrapper
  • We'll add slots inside this div

Step 3: Add the Header Slot

What we're doing: Add a named slot for the header with styling.

Key concept: Named slots use name="header" and can have default content.

app.component('Card', { template: ` <div class="card"> <div class="card-header"> <slot name="header">Default Header</slot> </div> </div> ` });

Explanation:

  • <slot name="header"> - Named slot for header
  • Default Header - Default content (shown when no content provided)
  • <div class="card-header"> - Wrapper with styling (green background, white text)
  • Negative margins (margin: -20px -20px 15px -20px) extend header to card edges
ðŸ’¡ CSS Tip: Negative margins allow the header to extend to the card's edges, creating a seamless look!

Step 4: Add the Body Slot

What we're doing: Add a named slot for the body with padding.

Key concept: Body slot has padding for content spacing.

app.component('Card', { template: ` <div class="card"> <div class="card-header"> <slot name="header">Default Header</slot> </div> <div style="padding: 15px;"> <slot name="body">Default Body Content</slot> </div> </div> ` });

Explanation:

  • <slot name="body"> - Named slot for body content
  • Default Body Content - Default content
  • style="padding: 15px;" - Adds padding around body content
  • Body comes after header in the template

Step 5: Add the Footer Slot

What we're doing: Add a named slot for the footer with styling.

Key concept: Footer has light background and border-top.

app.component('Card', { template: ` <div class="card"> <div class="card-header"> <slot name="header">Default Header</slot> </div> <div style="padding: 15px;"> <slot name="body">Default Body Content</slot> </div> <div class="card-footer"> <slot name="footer">Default Footer</slot> </div> </div> ` });

Explanation:

  • <slot name="footer"> - Named slot for footer
  • Default Footer - Default content
  • <div class="card-footer"> - Wrapper with styling (light background, border-top)
  • Negative margins extend footer to card edges

Step 6: Use the Component with Named Slots

What we're doing: Pass content to the Card component using named slots.

Key concept: Use <template #slot-name> to pass content.

<card> <template #header> <h3>Product Title</h3> </template> <template #body> <p>Product description</p> <p>\$29.99</p> </template> <template #footer> <button>Buy Now</button> </template> </card>

Explanation:

  • <template #header> - Passes content to header slot
  • # is shorthand for v-slot:
  • <template #body> - Passes content to body slot
  • <template #footer> - Passes content to footer slot
  • Content inside template tags replaces default slot content
ðŸ’¡ Shorthand: Use #header instead of v-slot:header - it's shorter and cleaner!

Step 7: See Default Content in Action

What we're doing: Use the component without providing content to see default slots.

Key concept: Default content shows when no content is provided.

<!-- With custom content --> <card> <template #header> <h3>Custom Header</h3> </template> <template #body> <p>Custom body content</p> </template> </card> <!-- Without content (shows defaults) --> <card></card>

Explanation:

  • When you provide content, it replaces the default
  • When you don't provide content, default content is shown
  • This makes the component flexible - works with or without content!
