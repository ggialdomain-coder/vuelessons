üéì Theory: What are Slots?

Why Slots?

Slots allow you to pass HTML content to components, making them more flexible and reusable!

Slots vs Props:

  • Props: Pass data (strings, numbers, objects)
  • Slots: Pass HTML content (templates, components)

Basic Slot Syntax:

// Component with slot
app.component('Card', {
    template: `
        <div class="card">
            <slot></slot>
        </div>
    `
});

// Use component with content
<card>
    <h3>My Title</h3>
    <p>My content goes here</p>
</card>

Named Slots:

// Component with named slots
template: `
    <div class="card">
        <slot name="header"></slot>
        <slot name="body"></slot>
        <slot name="footer"></slot>
    </div>
`

// Use with named slots
<card>
    <template v-slot:header>
        <h3>Header Content</h3>
    </template>
    <template v-slot:body>
        <p>Body Content</p>
    </template>
</card>

Slot Shorthand:

  • <template v-slot:header> or <template #header>
  • # is shorthand for v-slot:

Default Content:

<slot>Default content if nothing is passed</slot>
