Goal: Learn techniques to optimize Vue component performance. This tutorial covers v-show vs v-if, computed properties, and v-memo to prepare you for optimizing a slow component in the exercise.

Step 1: v-show vs v-if

v-if removes elements from DOM. v-show hides them with CSS. Choose based on toggle frequency.

// v-if - Removes from DOM (expensive to toggle) <div v-if="show">Content</div> // v-show - Hides with CSS (cheap to toggle) <div v-show="show">Content</div>
Code Explanation:
  • v-if="show" - Conditionally renders element (adds/removes from DOM)
  • Use when: Element rarely toggles, initial render cost is high
  • v-show="show" - Always renders, toggles CSS display property
  • Use when: Element frequently toggles, want to preserve state
  • Rule: Frequent toggles = v-show, Rare toggles = v-if

Step 2: Computed vs Methods in Templates

Use computed for derived data that's used in templates. Methods run every render, computed caches results.

// ‚ùå BAD - Method runs on every render <div>{{ expensiveCalculation() }}</div> // ‚úÖ GOOD - Computed caches result <div>{{ expensiveValue }}</div> setup() { const data = ref([1, 2, 3, 4, 5]); // Method - runs every time const expensiveCalculation = () => { return data.value.reduce((sum, n) => sum + n, 0); }; // Computed - caches result, only recalculates when data changes const expensiveValue = computed(() => { return data.value.reduce((sum, n) => sum + n, 0); }); return { expensiveValue }; }
Code Explanation:
  • expensiveCalculation() - Method runs on every render (even if data didn't change)
  • computed(() => {...}) - Only recalculates when dependencies change
  • Computed properties cache results for better performance
  • Use computed for values derived from reactive data

Step 3: Using v-memo for Expensive Lists

v-memo memoizes list items, preventing re-renders when dependencies haven't changed.

<div v-for="item in items" :key="item.id" v-memo="[item.id, item.name]"> <ExpensiveComponent :item="item" /> </div>
Code Explanation:
  • v-memo="[item.id, item.name]" - Memoizes based on these values
  • Only re-renders item if id or name changes
  • Useful for expensive list items with complex rendering
  • Improves performance with large lists

Step 4: Key Best Practices

1. Use :key in v-for

// ‚úÖ Always use :key <div v-for="item in items" :key="item.id">...</div>

2. Avoid inline functions in templates

// ‚ùå BAD <button @click="items.push({id: Date.now()})">Add</button> // ‚úÖ GOOD <button @click="addItem">Add</button>

3. Use Object.freeze() for static data

const staticData = Object.freeze([ { id: 1, name: 'Item 1' }, { id: 2, name: 'Item 2' } ]);
