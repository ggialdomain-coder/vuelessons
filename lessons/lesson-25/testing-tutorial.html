Goal: Learn testing concepts and how to test Vue components to ensure they work correctly.

Step 1: Why Test Components?

Testing ensures your components work correctly and prevents bugs.

Benefits:

  • Catches bugs before users do
  • Confidence when refactoring
  • Documents how components should work
  • Prevents regressions (old bugs coming back)

Step 2: Types of Tests

1. Unit Tests

Test individual components in isolation.

// Test a single component test('Counter increments', () => { const wrapper = mount(Counter); wrapper.find('button').trigger('click'); expect(wrapper.text()).toContain('Count: 1'); });

2. Integration Tests

Test how multiple components work together.

3. E2E Tests

Test complete user flows in a real browser.

Step 3: What to Test

Test the important behaviors of your component:

  • ‚úÖ Component renders correctly
  • ‚úÖ User interactions work (clicks, input)
  • ‚úÖ Props are handled correctly
  • ‚úÖ Events are emitted
  • ‚úÖ Computed properties calculate correctly
  • ‚úÖ Conditional rendering works

Step 4: Manual Testing Checklist

Even without automated tests, you can manually test components:

  1. Does the component render?
  2. Do buttons work when clicked?
  3. Does input update the component?
  4. Are props displayed correctly?
  5. Do events fire when expected?
  6. Does conditional rendering work?
  7. Are edge cases handled?

Step 5: Testing Concepts

Arrange-Act-Assert Pattern

// Arrange - Set up the test const wrapper = mount(MyComponent, { props: { initialCount: 5 } }); // Act - Perform the action wrapper.find('button').trigger('click'); // Assert - Check the result expect(wrapper.text()).toContain('Count: 6');
Code Explanation:
  • Arrange: Set up component with initial state
  • Act: Perform the action you're testing
  • Assert: Verify the expected outcome

Step 6: Testing Tools Overview

Vitest (Recommended for Vue 3)

// Install: npm install -D vitest @vue/test-utils import { mount } from '@vue/test-utils' import { describe, it, expect } from 'vitest' import MyComponent from './MyComponent.vue' describe('MyComponent', () => { it('renders correctly', () => { const wrapper = mount(MyComponent); expect(wrapper.text()).toContain('Hello'); }); });

Jest (Alternative)

Popular testing framework, works with Vue Test Utils.

Step 7: Common Test Patterns

Test Component Renders

test('renders component', () => { const wrapper = mount(MyComponent); expect(wrapper.exists()).toBe(true); });

Test User Interaction

test('button click increments count', () => { const wrapper = mount(Counter); const button = wrapper.find('button'); button.trigger('click'); expect(wrapper.vm.count).toBe(1); });

Test Props

test('displays prop value', () => { const wrapper = mount(MyComponent, { props: { message: 'Hello' } }); expect(wrapper.text()).toContain('Hello'); });

Step 8: Writing Test Cases

For each component, write test cases covering:

  1. Happy Path: Normal usage works
  2. Edge Cases: Empty data, null values, boundaries
  3. Error Cases: Invalid input, missing props
  4. User Interactions: Clicks, input, form submission
