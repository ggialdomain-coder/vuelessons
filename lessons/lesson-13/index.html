üéì Theory: Lists & Keys

What are Keys?

Keys are unique identifiers that help Vue track which items have changed, been added, or removed in a list.

Why Keys are Important:

  • ‚úÖ Help Vue efficiently update the DOM
  • ‚úÖ Prevent rendering bugs when list order changes
  • ‚úÖ Maintain component state correctly
  • ‚úÖ Improve performance with large lists

Basic Syntax:

// Using v-for with :key
<li v-for="item in items" :key="item.id">
    {{ item.name }}
</li>

// Using index as key (not recommended for dynamic lists)
<li v-for="(item, index) in items" :key="index">
    {{ item.name }}
</li>

Best Practices:

  • ‚úÖ Always use :key with v-for
  • ‚úÖ Use unique, stable identifiers (like id)
  • ‚úÖ Avoid using index as key when items can be reordered
  • ‚úÖ Keys should be unique within the list

What Happens Without Keys?

Without keys, Vue may incorrectly reuse DOM elements, causing:

  • ‚ùå Incorrect component state preservation
  • ‚ùå Performance issues
  • ‚ùå Bugs when list order changes
