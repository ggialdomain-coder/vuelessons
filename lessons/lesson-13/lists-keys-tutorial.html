Step 1: Understanding Keys

Goal: Build a todo list where you can add, delete, and toggle todos. Learn why keys are essential!

What are Keys?

  • ‚úÖ Unique identifiers for list items
  • ‚úÖ Help Vue track which items changed
  • ‚úÖ Required when using v-for
  • ‚úÖ Prevent bugs when list order changes
‚ö†Ô∏è Critical Rule: Always use :key with v-for! Without keys, Vue can't properly track list items, causing bugs and performance issues.

Why Keys Matter:

  • When you delete a todo, Vue needs to know which one to remove
  • When you reorder todos, Vue needs to track which item moved where
  • Keys help Vue efficiently update the DOM
  • Without keys, Vue might reuse the wrong DOM elements!

Step 2: Set Up Data Structure

What we're doing: Create the todos array with initial todos. Each todo needs an id, text, and completed status.

Key concept: Each todo must have a unique id - this will be our key!

data() { return { todos: [ { id: 1, text: 'Learn Vue', completed: false }, { id: 2, text: 'Build apps', completed: false } ], newTodo: '' } }

Explanation:

  • todos - Array of todo objects
  • Each todo has: id (unique identifier), text (todo text), completed (boolean)
  • id is critical - we'll use it as the key!
  • newTodo - Stores the text for new todos being added
üí° ID Generation: For new todos, use Date.now() to generate unique IDs. This ensures each todo has a unique identifier.

Step 3: Render Todos with v-for and :key

What we're doing: Use v-for to render todos, and always include :key!

Key concept: The :key must be unique and stable (use todo.id, not index!).

<ul class="todo-list"> <li v-for="todo in todos" :key="todo.id" class="todo-item"> {{ todo.text }} </li> </ul>

Explanation:

  • v-for="todo in todos" - Loops through todos array
  • :key="todo.id" - CRITICAL! Uses todo's unique ID as key
  • Vue uses the key to track which item is which
  • When you delete or reorder, Vue knows exactly which DOM element to update
  • {{ todo.text }} - Displays the todo text
‚ö†Ô∏è Never Forget :key! Always include :key="todo.id" when using v-for. Without it, you'll get bugs when adding, deleting, or reordering items!

Step 4: Add Checkbox with v-model and Toggle Method

What we're doing: Add a checkbox to toggle todo completion. We'll use v-model for binding, and also create a toggleTodo method as required by the exercise.

Key concept: Use v-model to bind checkbox to todo.completed, and add :class to style completed todos.

<li v-for="todo in todos" :key="todo.id" :class="{ completed: todo.completed }" class="todo-item"> <input type="checkbox" v-model="todo.completed"> <span>{{ todo.text }}</span> </li>

Explanation:

  • v-model="todo.completed" - Two-way binding to todo's completed status
  • When checkbox is checked, todo.completed becomes true
  • When unchecked, todo.completed becomes false
  • :class="{ completed: todo.completed }" - Adds 'completed' class when todo is done
  • This styles completed todos (line-through, opacity)
// Also create toggleTodo method (as required by exercise) methods: { toggleTodo(id) { const todo = this.todos.find(t => t.id === id); if (todo) { todo.completed = !todo.completed; } } }

Note: The exercise requires a toggleTodo(id) method. You can use v-model (which automatically toggles), or call @click="toggleTodo(todo.id)" instead. Both work, but v-model is simpler!

Step 5: Add Delete Functionality

What we're doing: Add a delete button and method to remove todos.

Key concept: Use filter() to remove the todo with matching ID.

<li v-for="todo in todos" :key="todo.id" :class="{ completed: todo.completed }" class="todo-item"> <input type="checkbox" v-model="todo.completed"> <span>{{ todo.text }}</span> <button @click="deleteTodo(todo.id)">Delete</button> </li> // In methods: deleteTodo(id) { this.todos = this.todos.filter(todo => todo.id !== id); }

Explanation:

  • @click="deleteTodo(todo.id)" - Calls deleteTodo with the todo's ID
  • filter() - Creates new array with todos that don't match the ID
  • todo.id !== id - Keeps todos whose ID doesn't match (removes the one that matches)
  • Because we used :key="todo.id", Vue knows exactly which DOM element to remove!
üí° Why Keys Help Here: When you delete a todo, Vue uses the key to find the exact DOM element to remove. Without keys, Vue might remove the wrong element!

Step 6: Add New Todo Functionality

What we're doing: Create a form to add new todos with unique IDs.

Key concept: Generate unique IDs for new todos (use Date.now()).

<div class="todo-form"> <input type="text" v-model="newTodo" placeholder="Add new todo"> <button @click="addTodo">Add</button> </div> // In methods: addTodo() { if (this.newTodo.trim()) { this.todos.push({ id: Date.now(), text: this.newTodo, completed: false }); this.newTodo = ''; } }

Explanation:

  • v-model="newTodo" - Binds input to newTodo data property
  • @click="addTodo" - Calls addTodo method when button clicked
  • this.newTodo.trim() - Checks if input is not empty (removes whitespace)
  • id: Date.now() - Generates unique ID using current timestamp
  • this.todos.push() - Adds new todo to the array
  • this.newTodo = '' - Clears the input after adding
üí° Unique IDs: Date.now() returns milliseconds since 1970, ensuring each todo gets a unique ID. This is perfect for keys!

Step 7: Add Completed Count (Computed Property)

What we're doing: Create a computed property to count completed todos and display stats.

Key concept: Use computed property to automatically calculate completed count.

// In computed: computed: { completedCount() { return this.todos.filter(todo => todo.completed).length; } } // In template: <div class="stats"> Completed: {{ completedCount }} / Total: {{ todos.length }} </div>

Explanation:

  • completedCount() - Computed property that recalculates automatically
  • filter(todo => todo.completed) - Gets only completed todos
  • .length - Counts how many completed todos
  • Computed property updates automatically when todos change!
  • Displays "Completed: X / Total: Y" in the template
