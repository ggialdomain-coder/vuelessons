Step 1: Understanding Dynamic Styling

Goal: Build a notification system where each notification has different styling based on its type (success, error, warning, info).

What is Dynamic Styling?

  • ‚úÖ Apply CSS classes conditionally using :class
  • ‚úÖ Apply inline styles dynamically using :style
  • ‚úÖ Change appearance based on component data
  • ‚úÖ Create interactive, responsive UIs

Two Main Approaches:

  • :class - Bind CSS classes (better for predefined styles)
  • :style - Bind inline styles (better for dynamic values like colors)

Step 2: Set Up Data Structure

What we're doing: Create the notifications array. Each notification needs an id, message, and type.

Key concept: The type will determine the styling!

data() { return { notifications: [] } }

Explanation:

  • notifications - Array to store all notifications
  • Each notification will have: id (unique identifier), message (text to display), type (success, error, warning, info)
  • We start with an empty array - notifications will be added dynamically

Step 3: Create Methods - addNotification

What we're doing: Create a method to add new notifications with unique IDs.

Key concept: Generate unique IDs and push new notifications to the array.

methods: { addNotification(type, message) { this.notifications.push({ id: Date.now(), message: message, type: type }); } }

Explanation:

  • addNotification(type, message) - Method that takes type and message as parameters
  • Date.now() - Generates unique ID using current timestamp
  • this.notifications.push() - Adds new notification to the array
  • Each notification object has: id, message, type
üí° Notification Types: Use 'success', 'error', 'warning', or 'info' as type values.

Step 4: Create Methods - removeNotification

What we're doing: Create a method to remove notifications by ID.

Key concept: Use filter() to create a new array without the notification to remove.

methods: { addNotification(type, message) { this.notifications.push({ id: Date.now(), message: message, type: type }); }, removeNotification(id) { this.notifications = this.notifications.filter(n => n.id !== id); } }

Explanation:

  • removeNotification(id) - Method that takes notification ID as parameter
  • filter(n => n.id !== id) - Creates new array with all notifications except the one with matching ID
  • This removes the notification from the list

Step 5: Create Methods - getColor (for :style)

What we're doing: Create a helper method that returns color based on notification type. This will be used with :style binding.

Key concept: Return different colors for different notification types.

methods: { addNotification(type, message) { this.notifications.push({ id: Date.now(), message: message, type: type }); }, removeNotification(id) { this.notifications = this.notifications.filter(n => n.id !== id); }, getColor(type) { const colors = { success: '#4caf50', error: '#f44336', warning: '#ff9800', info: '#2196f3' }; return colors[type] || '#666'; } }

Explanation:

  • getColor(type) - Method that returns color string based on type
  • colors object - Maps each type to its color
  • colors[type] - Gets color for the given type
  • || '#666' - Default color if type doesn't match
üí° Alternative: You can also use a computed property or inline the colors directly in :style binding.

Step 6: Add Buttons to Create Notifications

What we're doing: Add buttons in the HTML template that call addNotification with different types and messages.

Key concept: Each button triggers a different notification type. This matches the exercise requirement to "Add buttons to create different notification types".

<div class="notification-controls"> <button class="btn-success" @click="addNotification('success', 'Operation successful!')"> Success </button> <button class="btn-error" @click="addNotification('error', 'Something went wrong!')"> Error </button> <button class="btn-warning" @click="addNotification('warning', 'Please check your input!')"> Warning </button> <button class="btn-info" @click="addNotification('info', 'Here is some information!')"> Info </button> </div>

Explanation:

  • Four buttons, one for each notification type (success, error, warning, info)
  • @click="addNotification('success', '...')" - Calls method with type and message
  • Each button has a different CSS class (btn-success, btn-error, etc.) for styling
  • Clicking a button adds a new notification to the array
  • This matches the exercise requirement: "Add buttons to create different notification types"

Step 7: Render Notifications with v-for, :key, and :class

What we're doing: Use v-for to render notifications, always include :key, and use :class to apply CSS classes based on type.

Key concept: Use object syntax with :class to conditionally apply classes. This matches the exercise requirement: "Use :class with object syntax to apply CSS classes based on notification type".

<div class="notification-list"> <div v-for="notification in notifications" :key="notification.id" :class="{ success: notification.type === 'success', error: notification.type === 'error', warning: notification.type === 'warning', info: notification.type === 'info' }" class="notification"> {{ notification.message }} </div> </div>

Explanation:

  • v-for="notification in notifications" - Loops through notifications array (matches exercise requirement)
  • :key="notification.id" - Always use key with v-for! (matches exercise requirement: "don't forget :key!")
  • :class="{ success: notification.type === 'success', error: notification.type === 'error', warning: notification.type === 'warning', info: notification.type === 'info' }" - Object syntax for conditional classes
  • If notification.type === 'success' is true, the success class is applied
  • Same logic for error, warning, and info types
  • class="notification" - Base class always applied
  • This matches the exercise requirement: "Use :class with object syntax to apply CSS classes based on notification type"
üí° Object Syntax: :class="{ className: condition } applies className only when condition is true.

Step 8: Add :style Binding for Dynamic Colors

What we're doing: Add :style binding to apply dynamic background colors. This works alongside :class.

Key concept: Use object syntax with :style to set inline styles dynamically. This matches the exercise requirement: "Use :style with object syntax to apply dynamic colors based on type".

<div v-for="notification in notifications" :key="notification.id" :class="{ success: notification.type === 'success', error: notification.type === 'error', warning: notification.type === 'warning', info: notification.type === 'info' }" :style="{ backgroundColor: getColor(notification.type), color: 'white' }" class="notification"> {{ notification.message }} </div>

Explanation:

  • :style="{ backgroundColor: getColor(notification.type), color: 'white' }" - Object syntax for inline styles
  • getColor(notification.type) - Calls method to get color based on type (uses the getColor method we created in Step 5)
  • backgroundColor - CSS property (camelCase in Vue)
  • color: 'white' - Always white text
  • You can use either :class or :style, or both together!
  • This matches the exercise requirement: "Use :style with object syntax to apply dynamic colors based on type"
üí° CSS Properties: In :style, use camelCase (e.g., backgroundColor) or kebab-case with quotes (e.g., 'background-color').

Step 9: Add Close Button to Each Notification

What we're doing: Add a close button to each notification that calls removeNotification.

Key concept: Each notification should be dismissible. This matches the exercise requirement: "Each notification should have a close button to dismiss it".

<div v-for="notification in notifications" :key="notification.id" :class="{ success: notification.type === 'success', error: notification.type === 'error', warning: notification.type === 'warning', info: notification.type === 'info' }" :style="{ backgroundColor: getColor(notification.type), color: 'white' }" class="notification"> <span>{{ notification.message }}</span> <button class="notification-close" @click="removeNotification(notification.id)"> √ó </button> </div>

Explanation:

  • Wrapped message in <span> for layout (to separate message from close button)
  • @click="removeNotification(notification.id)" - Calls method with notification's ID (uses the removeNotification method we created in Step 4)
  • √ó - Close symbol (you can use 'X' or any icon)
  • Clicking the button removes that specific notification from the array
  • This matches the exercise requirement: "Each notification should have a close button to dismiss it"

üéì Key Concepts You Learned

:class Binding:

  • ‚úÖ Use object syntax: :class="{ className: condition }"
  • ‚úÖ Apply classes conditionally based on data
  • ‚úÖ Can combine multiple classes in one object
  • ‚úÖ Better for predefined CSS classes

:style Binding:

  • ‚úÖ Use object syntax: :style="{ property: value }"
  • ‚úÖ Apply inline styles dynamically
  • ‚úÖ Use camelCase for CSS properties (e.g., backgroundColor)
  • ‚úÖ Better for dynamic values (colors, sizes)

Notification System Pattern:

  • ‚úÖ Each notification has: id, message, type
  • ‚úÖ Use v-for with :key to render list
  • ‚úÖ Use :class or :style to style based on type
  • ‚úÖ Create methods: addNotification(type, message), removeNotification(id)
  • ‚úÖ Use filter() to remove items from array
