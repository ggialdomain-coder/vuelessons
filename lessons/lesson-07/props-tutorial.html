Learn how to pass data to components using props! This makes your components reusable with different data.

Step 1: Understanding the Problem

Problem: In Lesson 6, each ProductCard component had the same hardcoded data. We want to display different products!

Solution: Use props to pass data from parent to child component.

Props vs Data:

  • ‚úÖ Props - Data passed FROM parent TO child (one-way, read-only)
  • ‚úÖ Data - Data owned BY the component itself (can be modified)
üí° Analogy: Think of props like function parameters. You pass different values each time you call the function (use the component).

Step 2: Define Props in Component

What we're doing: Tell the component what props it will receive.

Key concept: Define props in the props option as an array or object.

app.component('ProductCard', { props: ['name', 'price', 'description'], template: `...`, // ... });

Explanation:

  • props: ['name', 'price', 'description'] - Array of prop names
  • Each prop name becomes available in the template and methods
  • Access props like data: {{ name }} in template, this.name in methods
‚ö†Ô∏è Important: Props are read-only! Don't try to modify them in the child component. They flow down from parent to child only.

Step 3: Use Props in Template

What we're doing: Use the props in the component template instead of hardcoded data.

Key concept: Access props the same way you access data - with {{ }} in templates.

app.component('ProductCard', { props: ['name', 'price', 'description'], template: ` <div class="card"> <h3>{{ name }}</h3> <p class="price">${{ price.toFixed(2) }}</p> <p>{{ description }}</p> <button @click="buy">Buy Now</button> </div> `, methods: { buy() { alert(`Added ${this.name} to cart!`); } } });

Explanation:

  • Use {{ name }} instead of {{ product.name }}
  • Use {{ price }} instead of {{ product.price }}
  • In methods, use this.name instead of this.product.name
  • No need for data() anymore - props replace it!

Step 4: Pass Props from Parent

What we're doing: Pass data to the component using props in the parent template.

Key concept: Use prop-name="value" to pass props. Use : for dynamic values!

<!-- Static props (strings) --> <product-card name="Vue.js Book" price="29.99" description="Learn Vue.js!"> </product-card> <!-- Dynamic props (from parent data) --> <product-card :name="product.name" :price="product.price" :description="product.description"> </product-card>

Explanation:

  • name="Vue.js Book" - Passes the string "Vue.js Book"
  • :name="product.name" - Passes the value of product.name (dynamic)
  • : is shorthand for v-bind:
  • Always use : for numbers, booleans, and variables!
üí° Remember:
  • name="value" ‚Üí Always passes string "value"
  • :name="value" ‚Üí Passes the actual value (number, boolean, variable, etc.)
  • For numbers: :price="29.99" (not price="29.99" which would be a string!)

Step 5: Using Props with v-for

What we're doing: Use v-for to render multiple components with different props.

Key concept: Loop through an array and pass each item's properties as props.

// Parent component data data() { return { products: [ { id: 1, name: 'Vue.js Book', price: 29.99, description: 'Learn Vue.js!' }, { id: 2, name: 'React Guide', price: 34.99, description: 'Master React' }, { id: 3, name: 'Angular Tutorial', price: 39.99, description: 'Complete course' } ] } } // In template <product-card v-for="product in products" :key="product.id" :name="product.name" :price="product.price" :description="product.description"> </product-card>

Explanation:

  • v-for="product in products" - Loops through products array
  • :key="product.id" - Required for v-for (unique identifier)
  • :name="product.name" - Passes each product's name as prop
  • Each iteration creates a new component instance with different props

Step 6: Complete Working Example

Here's the complete code for using props:

<script> const { createApp } = Vue; const app = createApp({ data() { return { products: [ { id: 1, name: 'Vue.js Book', price: 29.99, description: 'Learn Vue.js!' }, { id: 2, name: 'React Guide', price: 34.99, description: 'Master React' }, { id: 3, name: 'Angular Tutorial', price: 39.99, description: 'Complete course' } ] } } }); // Component with props app.component('ProductCard', { props: ['name', 'price', 'description'], template: ` <div class="card"> <h3>{{ name }}</h3> <p class="price">${{ price.toFixed(2) }}</p> <p>{{ description }}</p> <button @click="buy">Buy Now</button> </div> `, methods: { buy() { alert(`Added ${this.name} to cart!`); } } }); app.mount('#app'); </script> <!-- In HTML --> <div id="app"> <product-card v-for="product in products" :key="product.id" :name="product.name" :price="product.price" :description="product.description"> </product-card> </div>

üéì Key Concepts Summary

Defining Props:

  • ‚úÖ props: ['name', 'price'] - Array syntax (simple)
  • ‚úÖ Define in component definition
  • ‚úÖ Access like data: {{ name }} in template, this.name in methods

Passing Props:

  • ‚úÖ name="value" - Static string
  • ‚úÖ :name="value" - Dynamic value (variable, number, boolean)
  • ‚úÖ :name="product.name" - From parent data

Important Rules:

  • ‚ö†Ô∏è Props are read-only - don't modify them!
  • ‚ö†Ô∏è Props flow down (parent ‚Üí child) only
  • ‚ö†Ô∏è Always use : for numbers, booleans, and variables
  • ‚ö†Ô∏è Use :key when using v-for with components
üí° Remember: Props make components reusable! One component definition, many different instances with different data.
