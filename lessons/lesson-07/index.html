üéì Theory: What are Props?

Why Props?

Props allow you to pass data from a parent component to a child component. This makes components reusable with different data!

Props vs Data:

  • Props: Data passed FROM parent TO child (one-way, read-only)
  • Data: Data owned BY the component itself (can be modified)

Props Syntax:

// 1. Define props in component
app.component('ProductCard', {
    props: ['name', 'price', 'description'],
    template: `...`
});

// 2. Pass props from parent
<product-card 
    name="Vue.js Book" 
    :price="29.99" 
    :description="bookDesc">
</product-card>

Prop Binding:

  • name="value" - Passes string "value"
  • :name="value" - Passes the value of variable/value (dynamic)
  • :name="29.99" - Passes number 29.99
  • :name="true" - Passes boolean true

Important Rules:

  • Props are read-only - don't modify them in child component
  • Props flow down (parent ‚Üí child), never up
  • Use v-bind or : for dynamic values
  • Props are validated - Vue will warn if wrong type is passed
