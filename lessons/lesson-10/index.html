What is a Lifecycle?

Every Vue component goes through a lifecycle: it's created, mounted to the DOM, updated when data changes, and eventually unmounted (removed).

Lifecycle Stages:

1. Created - Component instance is created (data is available, but DOM not yet mounted)
2. Mounted - Component is added to the DOM (can access DOM elements)
3. Updated - Component re-renders when data changes
4. Unmounted - Component is removed from the DOM

Common Lifecycle Hooks:

export default {
    created() {
        // Component created, data available
        // Good for: API calls, setup
        console.log('Component created');
    },
    mounted() {
        // Component mounted to DOM
        // Good for: DOM manipulation, timers, event listeners
        console.log('Component mounted');
    },
    updated() {
        // Component updated (data changed)
        // Good for: Reacting to data changes
        console.log('Component updated');
    },
    unmounted() {
        // Component removed
        // Good for: Cleanup (clear timers, remove listeners)
        console.log('Component unmounted');
    }
}

When to Use Each Hook:

  • created(): API calls, initialize data, setup before DOM
  • mounted(): DOM manipulation, start timers, add event listeners
  • updated(): React to data changes, log updates
  • unmounted(): Cleanup timers, remove event listeners, cancel requests

Important Notes:

  • Hooks are called automatically by Vue
  • You don't call them manually
  • Use them to run code at specific lifecycle stages
  • Always clean up in unmounted() to prevent memory leaks!
