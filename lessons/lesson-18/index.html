üéì Theory: Route Parameters & Query Strings

Route Parameters

Route parameters allow you to create dynamic routes. For example, /user/123 where 123 is a parameter.

Defining Route Parameters:

const routes = [
    { path: '/user/:id', component: User },
    { path: '/product/:id', component: Product },
    { path: '/post/:category/:slug', component: Post }
];

Accessing Parameters:

// In component
export default {
    mounted() {
        const userId = this.$route.params.id;
        console.log('User ID:', userId);
    }
}

Query Strings

Query strings are optional parameters in the URL: /search?q=vue&sort=price

Accessing Query Strings:

// URL: /search?q=vue&sort=price
this.$route.query.q      // 'vue'
this.$route.query.sort   // 'price'
this.$route.query        // { q: 'vue', sort: 'price' }

Using Parameters in router-link:

<router-link :to="'/user/' + userId">View User</router-link>
<router-link :to="{ path: '/user', params: { id: userId }}">View User</router-link>

<router-link :to="{ path: '/search', query: { q: 'vue', sort: 'price' }}">
    Search
</router-link>

Key Concepts:

  • Params: Required dynamic segments in the URL path
  • Query: Optional key-value pairs after ?
  • $route.params: Object containing route parameters
  • $route.query: Object containing query parameters
