üéì Theory: Component Events

Why Component Events?

Props flow down (parent ‚Üí child), but events flow up (child ‚Üí parent). Events allow child components to communicate with their parent!

Data Flow:

  • Props: Parent ‚Üí Child (one-way, read-only)
  • Events: Child ‚Üí Parent (child emits, parent listens)

Event Syntax:

// 1. Child component emits event
methods: {
    addToCart() {
        this.$emit('add-to-cart', this.product);
    }
}

// 2. Parent listens to event
<product-card 
    @add-to-cart="handleAddToCart"
    :name="product.name">
</product-card>

// 3. Parent handles event
methods: {
    handleAddToCart(product) {
        this.cart.push(product);
    }
}

Event Naming:

  • Use kebab-case for event names: 'add-to-cart'
  • Listen with @add-to-cart or v-on:add-to-cart
  • Emit with this.$emit('add-to-cart', data)

Passing Data with Events:

  • this.$emit('event-name', data) - Pass data to parent
  • Parent receives data as first parameter: handleEvent(data) { }
  • Can pass multiple arguments: this.$emit('event', arg1, arg2)

Important Rules:

  • Events flow up only (child ‚Üí parent)
  • Use events to notify parent of actions
  • Parent decides what to do with the event
  • Event names should be descriptive
