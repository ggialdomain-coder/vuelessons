üìñ Building a Shopping Cart with State Management

Goal: Build a shopping cart system where ProductList and CartDisplay components share the same cart state using a store.

Step 1: Understanding the Problem

Challenge: We need two components to share the same shopping cart data:

  • ProductList - displays products and adds them to cart
  • CartDisplay - shows cart items and allows removal

When you add an item in ProductList, it should immediately appear in CartDisplay. This requires shared state!

// Without state management - components can't share data <ProductList /> // Has its own cart <CartDisplay /> // Has its own cart (different!) // With state management - both use the same store <ProductList /> // Uses cartStore <CartDisplay /> // Uses cartStore (same data!)

Step 2: Create the Cart Store - Part 1 (State)

First, create the store structure with reactive state. We need:

  • items - array to store cart items
  • total - number to store total price
const { reactive } = Vue; // Create the cart store const cartStore = { // State - reactive data that components will share state: reactive({ items: [], // Array of cart items total: 0 // Total price of all items }) };
üí° Important: reactive() makes the state reactive, so Vue automatically updates all components when the state changes!

Step 3: Create the Cart Store - Part 2 (Methods)

Now add methods to update the cart state. We need:

  • addItem(product) - adds a product to cart
  • removeItem(itemId) - removes an item from cart
  • clearCart() - empties the cart
  • calculateTotal() - calculates and updates total price
const cartStore = { state: reactive({ items: [], total: 0 }), // Add product to cart addItem(product) { this.state.items.push({ id: Date.now(), // Unique ID for each item name: product.name, price: product.price }); this.calculateTotal(); // Update total after adding }, // Remove item from cart removeItem(itemId) { const index = this.state.items.findIndex(item => item.id === itemId); if (index > -1) { this.state.items.splice(index, 1); this.calculateTotal(); // Update total after removing } }, // Clear all items from cart clearCart() { this.state.items = []; this.state.total = 0; }, // Calculate total price calculateTotal() { this.state.total = this.state.items.reduce((sum, item) => { return sum + item.price; }, 0); } };
üí° Note: We use this.state to access the state, and always call calculateTotal() after adding or removing items.

Step 4: Create ProductList Component

Create a component that displays products and adds them to the cart when clicked.

app.component('ProductList', { template: ` <div class="component-box"> <h3>Products</h3> <div v-for="product in products" :key="product.id" class="product-item"> <div> <strong>{{ product.name }}</strong> - ${{ product.price }} </div> <button @click="addToCart(product)">Add to Cart</button> </div> </div> `, data() { return { products: [ { id: 1, name: 'Laptop', price: 999 }, { id: 2, name: 'Mouse', price: 25 }, { id: 3, name: 'Keyboard', price: 75 }, { id: 4, name: 'Monitor', price: 200 } ] } }, methods: { addToCart(product) { // Use store method to add item cartStore.addItem(product); } } });
üí° Key Points:
  • Component has its own products data
  • Uses v-for to display products
  • Calls cartStore.addItem() when button is clicked

Step 5: Create CartDisplay Component

Create a component that displays cart items from the store, shows total, and allows removal.

app.component('CartDisplay', { template: ` <div class="component-box"> <h3>Shopping Cart</h3> <!-- Show message when cart is empty --> <div v-if="store.items.length === 0"> <p>Cart is empty</p> </div> <!-- Show cart items when not empty --> <div v-else> <div v-for="item in store.items" :key="item.id" class="cart-item"> <div> {{ item.name }} - ${{ item.price }} </div> <button @click="removeFromCart(item.id)">Remove</button> </div> <div style="margin-top: 15px; padding-top: 15px; border-top: 2px solid #ddd;"> <p><strong>Total: ${{ store.total }}</strong></p> <button @click="clearCart">Clear Cart</button> </div> </div> </div> `, data() { return { // Access store state store: cartStore.state } }, methods: { removeFromCart(itemId) { cartStore.removeItem(itemId); }, clearCart() { cartStore.clearCart(); } } });
üí° Key Points:
  • Accesses store state via data() { return { store: cartStore.state } }
  • Uses v-if to show empty message
  • Uses v-for to display cart items
  • Calls store methods to remove items or clear cart

Create the main app and use both components. They will automatically share the same cart state!

Step 8: How It Works

1. Shared State

Both components access cartStore.state, so they see the same data.

2. Reactivity

When cartStore.addItem() modifies state.items, Vue automatically updates CartDisplay because it's watching store.items.

3. Store Methods

Components call store methods (like addItem, removeItem) instead of modifying state directly. This keeps the code organized.
