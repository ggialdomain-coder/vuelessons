Goal: Learn how to create a Modal component using Teleport to render content outside component hierarchy. This tutorial prepares you for the exercise where you'll build a modal system with Teleport.

Step 1: Understanding Teleport

Teleport allows you to render a component's content to a different location in the DOM, outside the component's hierarchy.

Why use it?

  • Modals need to be at body level (avoid z-index issues)
  • Tooltips and notifications
  • Breaking out of parent container constraints

Step 2: Basic Teleport Syntax

Wrap content in <Teleport> and specify where to render it with the to prop.

<Teleport to="body"> <div class="modal"> Modal content here </div> </Teleport>
Code Explanation:
  • <Teleport> - Vue component that teleports content
  • to="body" - Renders content as child of <body> element
  • Can use CSS selector: to="#modal-container" or to=".container"
  • Content is moved to target location but keeps Vue reactivity

Step 3: Create Modal Component with Teleport

Create a modal that uses Teleport to render at body level.

const Modal = { template: '' + '
' + '
' + '

Modal Title

' + '

This is a modal message

' + '' + '
' + '
' + '', props: { isOpen: Boolean }, emits: ['close'], setup(props, { emit }) { const close = () => { emit('close'); }; return { close }; } };
Code Explanation:
  • <Teleport to="body"> - Teleports modal to body element
  • v-if="isOpen" - Only shows modal when isOpen is true
  • class="overlay" - Full-screen dark background
  • @click="close" - Closes modal when clicking overlay
  • @click.stop - Prevents click from bubbling to overlay (stops propagation)
  • Modal renders at body level, not inside component hierarchy

Step 4: Using the Modal Component

Use the modal component in your app. Even though it's used inside a component, it renders at body level.

const app = createApp({ setup() { const showModal = ref(false); return { showModal }; } });
<!-- Register Modal component --> app.component('Modal', Modal); <!-- Template --> <div id="app"> <button @click="showModal = true">Open Modal</button> <Modal :isOpen="showModal" @close="showModal = false" /> </div>
Code Explanation:
  • Modal component is used inside the app
  • But it renders at <body> level due to Teleport
  • Props and events work normally
  • Vue reactivity is maintained

Step 6: Understanding Suspense

Suspense allows you to show fallback content while async components are loading.

Use Cases:

  • Async components that take time to load
  • Components that fetch data
  • Code-splitting scenarios

Step 7: Using Suspense

Wrap async components in <Suspense> and provide fallback content.

<Suspense> <template #default> <AsyncComponent /> </template> <template #fallback> <div>Loading...</div> </template> </Suspense>
Code Explanation:
  • <Suspense> - Wraps async component
  • <template #default> - The async component to load
  • <template #fallback> - Content shown while loading
  • Shows fallback until async component is ready
  • Then shows the actual component

Step 8: Creating Async Component

Create an async component that simulates loading.

// Async component (simulates loading) const AsyncComponent = { async setup() { // Simulate API call await new Promise(resolve => setTimeout(resolve, 2000)); const message = ref('Component loaded!'); return { message }; }, template: '

Async Component

{{ message }}

' };
Code Explanation:
  • async setup() - Makes setup function async
  • await new Promise(...) - Simulates delay (like API call)
  • Suspense will show fallback during this delay
  • Once resolved, component renders

Step 9: Complete Example

Complete example using both Teleport and Suspense.

const app = createApp({ setup() { const showModal = ref(false); return { showModal }; } }); // Register components app.component('Modal', { /* ... */ }); app.mount('#app');
<!-- HTML --> <div id="app"> <button @click="showModal = true">Open Modal</button> <Teleport to="body"> <Modal v-if="showModal" @close="showModal = false" /> </Teleport> <Suspense> <template #default> <AsyncComponent /> </template> <template #fallback> <div>Loading component...</div> </template> </Suspense> </div>

üéì Key Concepts You Learned

  1. Teleport: Renders content to different DOM location
  2. to prop: CSS selector or element reference for target
  3. Use Cases: Modals, tooltips, notifications
  4. Suspense: Shows fallback while async component loads
  5. #default: Slot for async component
  6. #fallback: Slot for loading content
  7. Reactivity: Both maintain Vue reactivity
