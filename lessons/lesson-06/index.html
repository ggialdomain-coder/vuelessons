üéì Theory: What are Components?

Why Components?

Components are reusable pieces of UI. Think of them like LEGO blocks - you build small pieces and combine them to create larger applications.

Benefits:

  • Reusability: Write once, use many times
  • Organization: Break complex apps into smaller pieces
  • Maintainability: Easier to find and fix bugs
  • Collaboration: Team members can work on different components

Component Structure:

// Define a component
const MyComponent = {
    template: `<div>Hello from Component!</div>`,
    data() {
        return {
            message: 'Hello'
        }
    }
}

// Register it
app.component('MyComponent', MyComponent)

// Use it in template
<MyComponent />

Component Options:

  • template - HTML structure
  • data() - Component's own data (must be a function!)
  • methods - Component methods
  • computed - Computed properties
  • props - Data passed from parent (we'll learn this in Lesson 7)

Important: data() must be a function!

// ‚úÖ CORRECT
data() {
    return {
        count: 0
    }
}

// ‚ùå WRONG
data: {
    count: 0
}

Why? Each component instance needs its own data. If it's an object, all instances would share the same data!
