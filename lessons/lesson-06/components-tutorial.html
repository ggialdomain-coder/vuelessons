Learn how to build a reusable Product Card component! This tutorial will guide you through creating exactly what you need for the exercise.

Step 1: Understanding What We're Building

Goal: Create a ProductCard component that displays product information and has a "Buy Now" button.

What the component needs:

  • Product name (heading)
  • Product price (formatted to 2 decimal places)
  • Product description
  • "Buy Now" button that shows an alert

Why components?

  • ‚úÖ Write once, use many times - create the component once, use it for multiple products
  • ‚úÖ Each instance is independent - each product card has its own data
  • ‚úÖ Easy to maintain - change the component once, all instances update

Step 2: Set Up the Vue App

What we're doing: Create the main Vue app that will hold our component.

Key concept: We need to create the app first, then register components to it.

<script src="https://unpkg.com/vue@3/dist/vue.global.js"></script> <script> const { createApp } = Vue; const app = createApp({ data() { return { message: 'Product Store' } } }); // We'll register our component here app.mount('#app'); </script>

Explanation:

  • createApp() - Creates a new Vue application instance
  • We pass an object with data() function (even if it's just a placeholder)
  • app.mount('#app') - Connects Vue to the HTML element with id="app"
  • We'll register components before mounting

Step 3: Register the ProductCard Component

What we're doing: Register the component with Vue using app.component().

Key concept: Component names use PascalCase, but we'll use kebab-case in templates.

app.component('ProductCard', { // Component definition will go here });

Explanation:

  • app.component() - Registers a component globally
  • First parameter: 'ProductCard' - Component name in PascalCase
  • Second parameter: Component definition object (we'll fill this in next steps)
  • After registration, we can use <product-card></product-card> in HTML
üí° Naming: Vue automatically converts PascalCase to kebab-case, so 'ProductCard' becomes <product-card> in templates.

Step 4: Add the Template

What we're doing: Create the HTML structure for the product card.

Key concept: The template property contains the HTML that will be rendered.

app.component('ProductCard', { template: ` <div class="card"> <h3>{{ product.name }}</h3> <p class="price">${{ product.price.toFixed(2) }}</p> <p>{{ product.description }}</p> <button @click="buy">Buy Now</button> </div> ` });

Explanation:

  • template - The HTML structure (use backticks for multi-line strings)
  • {{ product.name }} - Displays the product name (we'll add data next)
  • {{ product.price.toFixed(2) }} - Formats price to 2 decimal places
  • {{ product.description }} - Displays the description
  • @click="buy" - Calls the buy() method when button is clicked
üí° Template Syntax: Use double curly braces {{ }} to display data, and @click for button clicks.

Step 5: Add Component Data

What we're doing: Add the data() function to store product information.

Key concept: data() MUST be a function that returns an object!

app.component('ProductCard', { template: ` <div class="card"> <h3>{{ product.name }}</h3> <p class="price">${{ product.price.toFixed(2) }}</p> <p>{{ product.description }}</p> <button @click="buy">Buy Now</button> </div> `, data() { return { product: { name: 'Vue.js Book', price: 29.99, description: 'Learn Vue.js the easy way!' } } } });
‚ö†Ô∏è CRITICAL: data() must be a function, not an object!
‚úÖ data() { return { product: {...} } }
‚ùå data: { product: {...} }
Why? Each component instance needs its own data. If it's an object, all instances would share the same data!

Explanation:

  • data() - Function that returns an object with component data
  • product - Object containing name, price, and description
  • Each time you use <product-card>, it gets its own copy of this data
  • Access in template: {{ product.name }}
  • Access in methods: this.product.name

Step 6: Add the Buy Method

What we're doing: Add a method that shows an alert when the "Buy Now" button is clicked.

Key concept: Methods are functions defined in the methods option.

app.component('ProductCard', { template: ` <div class="card"> <h3>{{ product.name }}</h3> <p class="price">${{ product.price.toFixed(2) }}</p> <p>{{ product.description }}</p> <button @click="buy">Buy Now</button> </div> `, data() { return { product: { name: 'Vue.js Book', price: 29.99, description: 'Learn Vue.js the easy way!' } } }, methods: { buy() { alert(`Added ${this.product.name} to cart!`); } } });

Explanation:

  • methods - Object containing component methods
  • buy() - Method that runs when button is clicked
  • this.product.name - Access component data using this
  • alert() - Shows a browser alert with the product name
  • Template uses @click="buy" to call this method

<div id="app"> <h2>Our Products</h2> <div class="products-container"> <product-card></product-card> <product-card></product-card> <product-card></product-card> </div> </div>

Step 8: Complete Working Code

Here's the complete code for the ProductCard component:

<script src="https://unpkg.com/vue@3/dist/vue.global.js"></script> <script> const { createApp } = Vue; const app = createApp({ data() { return { message: 'Product Store' } } }); // Register ProductCard component app.component('ProductCard', { template: ` <div class="card"> <h3>{{ product.name }}</h3> <p class="price">${{ product.price.toFixed(2) }}</p> <p>{{ product.description }}</p> <button @click="buy">Buy Now</button> </div> `, data() { return { product: { name: 'Vue.js Book', price: 29.99, description: 'Learn Vue.js the easy way!' } } }, methods: { buy() { alert(`Added ${this.product.name} to cart!`); } } }); app.mount('#app'); </script>

üéì Key Concepts You Learned

  • ‚úÖ app.component('ComponentName', { ... }) - Register a component
  • ‚úÖ template - HTML structure of the component
  • ‚úÖ data() - Must be a function that returns an object
  • ‚úÖ methods - Functions that handle actions

Template Syntax:

  • ‚úÖ {{ variable }} - Display data
  • ‚úÖ {{ product.price.toFixed(2) }} - Format numbers
  • ‚úÖ @click="methodName" - Handle button clicks

Accessing Data:

  • ‚úÖ In template: {{ product.name }}
  • ‚úÖ In methods: this.product.name

Critical Rules:

  • ‚ö†Ô∏è data() must be a function (not an object!)
  • ‚ö†Ô∏è Component names use PascalCase: 'ProductCard'
  • ‚ö†Ô∏è Use kebab-case in templates: <product-card>
