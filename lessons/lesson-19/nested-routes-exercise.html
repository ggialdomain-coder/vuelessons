Your Task: Build a user dashboard with nested routes.

Part 1: Create Parent Layout

  1. Create a /user route with a UserDashboard component
  2. The component should:
    • Display "User Dashboard" heading
    • Have navigation links for child routes
    • Include <router-view> for child routes

Part 2: Create Child Routes

  1. Create nested routes:
    • /user/profile - UserProfile component
    • /user/settings - UserSettings component
    • /user/orders - UserOrders component
  2. Each child component should display relevant content

Part 3: Navigation

  1. Add navigation links in the parent component for all child routes
  2. Style the navigation (active link highlighting)
  3. Test navigation between nested routes

Part 4: Default Route

  1. Set UserProfile as the default child route (empty path)
  2. When visiting /user, it should show the profile

Part 5: Bonus - Multiple Levels

  1. Add a nested route under orders: /user/orders/:orderId
  2. Create an OrderDetail component
  3. Display order ID from route parameter

üí° Hints

  • Use children array in route definition
  • Child paths are relative (no leading /)
  • Parent component MUST have <router-view>
  • Use empty path '' for default child route
  • Full paths are: /user/profile, /user/settings, /user/orders
